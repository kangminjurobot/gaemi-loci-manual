## What is GAEMI Loci?

For an autonomous mobile robot to drive safely outdoors, it needs a precise High-Definition (HD) map.
**GAEMI Loci** is a **web platform** that turns data collected in the field by a robot into an HD map, lets you edit it, and deploys the result back to the robot.

<Aside type="tip">
GAEMI Loci runs entirely in the browser, so no additional software installation is required.
</Aside>

GAEMI Loci organizes HD map generation into **three Phases**.
Each Phase is broken down into individual Steps, and you can monitor current progress from the dashboard.

<img src={workflowPhases.src} alt="Diagram of the GAEMI Loci three-phase workflow" />

## Phase 1 — Preparation

<Image src={screenshotPhase1} alt="Phase 1 — setting up the service area on a satellite map and marking the robot's driving path." />

<YouTube id="p-k46jWUCjc" title="Phase 1 — Preparation Workflow" />

In this stage, you prepare from the office before deploying the robot to the field.

<Steps>
1. **Define the service area**: Mark the region where the robot will operate on a satellite map.
2. **Author the robot path**: Edit the driving path the robot will follow within the service area.
3. **Review and approve the path**: Have a reviewer inspect and approve the authored path.
</Steps>

<Aside type="tip">
Phase 1 can be carried out remotely without visiting the site.
</Aside>

## Phase 2 — On-Site Installation

<Image src={screenshotPhase2} alt="Phase 2 — configuring charging stations on-site and collecting mapping data with the robot." />

<YouTube id="--WVn8SYrYQ" title="Phase 2 — On-Site Installation Workflow" />

In this stage, you deploy the robot to the field, collect real-world mapping data, and generate the HD map.

<Steps>
1. **Collect mapping data**: Drive the robot around the site to gather data about the surrounding environment.
2. **Upload data**: Upload the collected data to the server.
3. **Automated HD map generation**: The server analyzes the uploaded data and automatically generates an HD map.
4. **Edit and refine the map**: Review the generated map in the web interface and apply any necessary edits or corrections.
5. **Final upload**: Upload the completed map in a format ready for deployment to the robot.
</Steps>

## Phase 3 — Expansion

<Image src={screenshotPhase3} alt="Phase 3 — monitoring autonomous driving status and expanding the map from the map editing and control dashboard." />

<YouTube id="jOmIDXuJR1g" title="Phase 3 — Expansion Workflow" />

In this stage, you expand the service area or improve an existing map. It can be repeated as needed.

<Steps>
1. **Map additional areas**: Collect additional data from new regions.
2. **Merge with the existing map**: Integrate the newly collected data into the existing HD map.
3. **Iterative refinement**: Continuously improve map quality by adding data session by session.
</Steps>

<Aside>
Phase 3 can be performed even while the service is running. Simply add a new session whenever the operating area is expanded.
</Aside>

## Map Generation at a Glance

Here is a brief summary of the entire process.

<Steps>
1. **Collect data in the field** — The robot drives around and scans the surrounding environment.
2. **Automatic upload to the server** — The collected data is transferred to the cloud server.
3. **Automated HD map generation** — The server processes the data to produce an HD map. You can monitor progress in real time.
4. **Edit the map in the browser** — Review the generated map in a web browser and perform tasks such as obstacle correction and path refinement.
5. **Deploy to the robot** — Deliver the completed map to the robot for use in autonomous driving.
</Steps>
